# Code lesen, testen und verbessern

## Ziel

Die Teilnehmer koennen KI-erzeugten Code ausprobieren, grob einordnen und gezielt verbessern lassen.

## Code lesen

Anfaenger muessen nicht jede Zeile sofort verstehen. Wichtig ist zuerst:

- Welche Dateien gibt es?
- Was ist HTML, CSS und JavaScript?
- Wo stehen sichtbare Texte?
- Wo stehen Farben, Abstaende und Layout?
- Wo passiert Logik?

## Testen im Browser

Prueffragen:

- Laesst sich die Datei oeffnen?
- Sind alle Texte sichtbar?
- Funktionieren Buttons und Eingaben?
- Ist die Seite auf dem Smartphone lesbar?
- Was passiert bei leeren oder falschen Eingaben?

## Fehlermeldungen nutzen

Eine gute Anfrage an die KI enthaelt:

- was man getan hat
- was erwartet wurde
- was tatsaechlich passiert ist
- die genaue Fehlermeldung
- den betroffenen Codeausschnitt

## Verbesserungen

Typische naechste Schritte:

- Texte vereinfachen
- Bedienung klarer machen
- Farben und Kontraste verbessern
- Eingaben pruefen
- doppelten Code entfernen
- Kommentare ergaenzen

## Merksatz

Nicht der erste Code ist entscheidend, sondern ob man ihn pruefen und verbessern kann.
