# Deployment: Server, VPS und Cloud-Instanz

## Ziel

Die Teilnehmer verstehen, wie aus einem lokal lauffaehigen Projekt eine erreichbare Anwendung werden kann. Je nach Vorkenntnis und vorhandenem Zugang kann ein Deployment praktisch vorbereitet oder als Ausblick besprochen werden.

## Moegliche Ziele

### Lokales Projekt

Geeignet fuer:

- erste Tests
- private Nutzung
- Projekte ohne Veroeffentlichung

Vorteile:

- keine Zusatzkosten
- keine Serverpflege
- geringes Risiko

### GitHub oder statisches Hosting

Geeignet fuer:

- einfache Webseiten
- Kurs- und Projektseiten
- Portfolios
- kleine Tools ohne Serverlogik

Vorteile:

- guter Einstieg in Veroeffentlichung
- Projektstand ist versioniert
- kein eigener Serverbetrieb noetig

### Eigener Server oder VPS

Geeignet fuer:

- eigene Kontrolle ueber Umgebung und Daten
- kleine Webanwendungen
- Experimente mit Linux, Docker und Diensten

Voraussetzungen:

- Zugangsdaten sicher verwahren
- Grundverstaendnis fuer SSH oder Server-Konsole
- Updates und Sicherheit ernst nehmen
- keine sensiblen Echtdaten fuer Kursuebungen verwenden

### Cloud-Instanz

Geeignet fuer:

- temporaere Testumgebungen
- skalierbare spaetere Projekte
- Lernpfade Richtung Cloud-Betrieb

Zu beachten:

- Kostenkontrolle
- Zugangsschluessel und Rechte
- Datenschutz und Speicherort
- Dienste nach dem Kurs wieder stoppen, wenn sie nicht gebraucht werden

## Kursregel

Ein eigener Server, VPS oder eine Cloud-Instanz ist optional. Niemand soll im Kurs unter Zeitdruck produktive Zugangsdaten oder echte Kundendaten verwenden.

## Sicherer Ablauf

1. Lokal lauffaehige Version erstellen.
2. Projektstruktur aufraeumen.
3. Geheimnisse und private Daten entfernen.
4. Projektstand mit Git sichern.
5. Deployment-Ziel auswaehlen.
6. Nur eine kleine, unkritische Demo veroeffentlichen.
7. Nach dem Kurs pruefen, ob Dienste weiterlaufen sollen.

## Merksatz

Veroeffentlichen ist ein eigener Arbeitsschritt. Ein Coding-Agent kann helfen, aber Betrieb, Sicherheit, Kosten und Zugangsdaten bleiben Verantwortung des Menschen.
