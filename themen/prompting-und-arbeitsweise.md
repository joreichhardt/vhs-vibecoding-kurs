# Prompting und Arbeitsweise

## Ziel

Die Teilnehmer koennen Codex-Auftraege so formulieren, dass brauchbare, pruefbare und verbesserbare Ergebnisse entstehen.

## Gute Codex-Auftraege enthalten

- Ziel: Was soll am Ende entstehen?
- Kontext: Fuer wen ist es gedacht?
- Eingaben: Welche Daten oder Beispiele gibt es?
- Grenzen: Was soll nicht passieren?
- Ausgabeformat: Text, Tabelle, Code, Checkliste oder Schrittfolge?
- Arbeitsbereich: Welche Dateien oder Ordner sind relevant?
- Fertig wenn: Woran soll Codex erkennen, dass die Aufgabe erledigt ist?
- Rueckfrage-Regel: Wann soll die KI nachfragen?

## Beispielstruktur

```text
Ich moechte eine einfache Webseite fuer einen VHS-Kurs erstellen.
Zielgruppe sind Einsteiger.
Die Seite soll eine Kursbeschreibung, Termine und eine Materialliste enthalten.
Bitte erstelle ein kleines Browser-Projekt mit index.html, style.css und script.js.
Verwende keine externen Bibliotheken und keine echten personenbezogenen Daten.
Fertig ist die Aufgabe, wenn ich index.html im Browser oeffnen und die Seite bedienen kann.
Erklaere danach kurz, welche Dateien du geaendert hast.
```

## Besser arbeiten in kleinen Schritten

Statt:

- "Baue mir eine perfekte App."

Besser:

- "Erstelle zuerst eine einfache HTML-Struktur."
- "Fuege danach ein Formular hinzu."
- "Pruefe den Code auf Fehler."
- "Erklaere mir, welche Teile ich anpassen muss."

## Nuetzliche Rueckmeldungen

- "Das funktioniert, aber der Button ist unklar beschriftet."
- "Bitte vereinfache den Code fuer Anfaenger."
- "Die Datei soll ohne Internetverbindung funktionieren."
- "Pruefe, ob personenbezogene Daten gespeichert werden."
- "Mach daraus eine Version fuer Smartphone-Bildschirme."
- "Arbeite nur in diesem Projektordner."
- "Aendere erst die Minimalversion, keine Zusatzfunktionen."

## Merksatz

Ein Prompt ist kein Zauberspruch, sondern ein Arbeitsauftrag. Gute Ergebnisse entstehen durch Kontext, Pruefung und Korrektur.
